# P0-2: Daily revenue chart fix

## Scope
Only `DailyRevenueCard.tsx` + its CSS in `sales-tab.css`. No new library; render with divs.

## Plan
- Remove per-column currency label row (`daily-revenue-col__amount`) — kills the truncated `S/ 0…` noise.
- Restructure layout: chart row becomes `[bars area | y-axis column]` so MAX/AVG/0 sit on the right edge.
- Y-axis: 3 labels (MAX, AVG, 0) absolutely positioned within the bars track at top/mid/bottom; `--font-mono`, `--text-xs`, `--color-text-tertiary`. MAX shows formatted currency of `max`, AVG shows avg of non-empty (or all 7) days.
- Bars: keep flex row, each col has a vertical track. Bar `background: var(--color-brand)`, today full opacity; prior days `--color-brand-tint` at 0.6 opacity. `border-radius: var(--radius-sm) var(--radius-sm) 0 0`. `min-height: 2px` so zero-days still show a hairline.
- Drop the empty-bar variant; min-height handles it.
- Keep day-of-week tick row below the track unchanged.
- Defer tap-to-tooltip (notes in report) — would need portal/popover infra, out of scope.
- i18n: add `sales.reports.daily_revenue_axis_max/avg/zero` to all 11 locale files; regen messageIds.d.ts.
- Verify in Playwright: navigate to sales tab, screenshot before/after.
